\# Prompting Patterns



Reusable patterns abstracted from successful prompts across sessions.



\## Purpose

Capture general approaches, not project-specific details.

Focus on what makes prompts effective and outcomes measurable.



\## Entry Format

```

---

Pattern Name: \[Descriptive title]

Category: \[Architecture/Planning/Debugging/Testing/Documentation/Research/Refactoring]

When to Use: \[Situation/problem description]



Template Structure:

```

\[CONTEXT: What's the current state]

\[REQUEST: Specific action with deliverables]

\[CONSTRAINTS: Technical/business requirements]

\[GUIDANCE: Must-haves, exclusions]

\[COLLABORATION: Invitation to clarify]

```



Key Elements:

\- \[Bullet: What makes this effective]

\- \[Bullet: Critical components]



Example Outcome: \[Generic result with metrics if applicable]

Example: "Updated module to cache results using LRU, achieved 166x speedup"



Sessions Using This: \[SESSION-IDs where pattern appeared]



Evolution: \[How pattern was refined across sessions - add as discovered]

---

```
